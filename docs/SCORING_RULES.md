# HR模拟器 评分规则文档（开发者参考）

> 本文档仅供开发者阅读，不在游戏内展示。

---

## 总分结构（满分100）

| 模块 | 满分 | 说明 |
|------|------|------|
| 简历筛选 | 20 | 正确识别候选人质量 |
| 面试反馈 | 50 | 三维度评分 + 通过/不通过决策 + 反问环节 |
| 谈薪环节 | 30 | Offer接受 + 谈判技巧 + 定薪策略 |

---

## 一、简历筛选（20分）

基础分 = 15

对每位候选人：

| 候选人质量 | 通过（进入面试） | 淘汰 |
|-----------|-----------------|------|
| 优质 (quality >= 7) | +5 | -3 |
| 普通 (quality 4-6) | 不变 | 不变 |
| 低质 (quality <= 3) | -3 | +3 |

最终简历分 = clamp(基础分 + 累计加减, 0, 20)

---

## 二、面试反馈（50分，对所有面试候选人取平均）

每位候选人的面试反馈分由三部分组成：

### 2a. 三维度评分（30分，每维度10分）

支持半星（0.5星精度），每维度与候选人 `idealStars` 对比：

| 偏差（半星为单位） | 得分 |
|-------------------|------|
| 精准命中（差0） | 10 |
| 差0.5星 | 8 |
| 差1星 | 6 |
| 差1.5星 | 4 |
| 差2星 | 2 |
| 差2.5星+ | 0 |

每维度最低0分。

### 2b. 通过/不通过决策（20分）

基础分 = 10，上限15，下限0

| 候选人质量 | 通过面试 | 不通过面试 |
|-----------|---------|-----------|
| 优质 (quality >= 7) | +10 | -5 |
| 普通 (quality 4-6) | +5 | 0 |
| 低质 (quality <= 3) | -5 | +5 |

注意：通过质量极差(quality<=3)的候选人会触发HRD警告弹窗，跳过谈薪直接进入HRD Review。

### 2c. 反问环节附加分

| 情况 | 加分 |
|------|------|
| 好回答（good） | +5 |
| 中回答（mid） | 不变 |
| 差回答（bad） | 不变 |
| 触发HRD来电质问 | -5（从面试反馈基础分中扣除） |

---

## 三、谈薪环节（30分）

仅对进入谈薪的候选人计算，然后取所有候选人平均。
未进入谈薪的候选人（面试不通过 / HRD警告跳过 / 简历淘汰）不参与谈薪分计算。

### 3a. Offer接受（10分）

基础分 = 5

| 候选人质量 | 接受Offer | 拒绝Offer |
|-----------|----------|----------|
| 优质 (quality >= 7) | +5 | -2 |
| 普通 (quality 4-6) | +3 | 0 |
| 低质 (quality <= 3) | 0 | 0 |

### 3b. 谈判技巧（15分）

基础分 = 0，对3轮谈判的每轮选择：

| 选项类型 | 加分 |
|---------|------|
| perfect（完美命中） | +5 |
| close（擦边） | +3 |
| irrelevant（无关） | +2 |
| negative（负面） | +0 |

满分15分（3轮 × 5分）。

### 3c. 定薪策略（5分）

基础分 = 3

| 最终定薪结果 | 加分 |
|------------|------|
| 命中候选人期望下限 | +2 |
| 低于候选人期望下限 | 额外+5（加到谈薪总分，但不超过单人30分上限） |

## 特殊规则

### HRD警告（通过低质候选人）
- 当玩家在面试反馈中选择"通过"quality<=3的候选人时
- 弹出HRD警告弹窗，提示该候选人存在明显问题
- 跳过谈薪环节，直接进入HRD Review
- 该候选人视为 `offerGiven = false`，记录 `hrdWarning = true`

### HRD来电质问
- 候选人反问环节连续两轮都选最差回答时触发
- 扣5分（从面试反馈分中扣除）
- 累计2次质问 → 强制结束游戏

### Offer状态区分
- `offerGiven: true` — 候选人接受了Offer
- `offerRejected: true` — 进入谈薪但候选人拒绝了Offer
- `interviewFailed: true` — 面试不通过（玩家选择不通过）
- `hrdWarning: true` — 通过低质候选人触发HRD警告
- `resumeRejected: true` — 简历筛选阶段被淘汰
