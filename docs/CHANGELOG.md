# HR模拟器 — 版本更新日志

---

## v1.4 (2026-04-14)

### Bug 修复
- **本地模式问答错配**：面试中多个不同问题出现相同回答。根因是问题来自通用题库、答案从独立回答池随机取，两者无映射关系。新增 `interviewQA` 问答绑定数据结构，问题和 good/mid/bad 三档回答严格配对
- **Q&A 匹配逻辑加固**：改为精确匹配优先，子串匹配要求问题文本≥8 字，防止短问题误匹配到无关 QA 对
- **HRD 场景定时器泄漏**：信件打字机效果、回复延迟等 6 处 `setTimeout`/`setInterval` 未注册到场景清理系统，场景切换后仍继续执行。全部用 `registerSceneTimer()` 包装
- **空值防御**：`finishCandidateInterview` 和 `enterHrdReviewAfterNegotiation` 增加候选人/评分数据的空值检查，防止异常状态下崩溃

### 数据更新
- 调用 DeepSeek API 生成 108 组面试问答数据（3 JD × 3 候选人档次 × 12 对），每对包含问题文本和 good/mid/bad 三种质量的回答
- 回答内容匹配候选人人设（如 bad 档张伟的奶茶/游戏梗、刘子轩的直播话术）

### 工具链
- `pre-build-check.sh` 新增 `interviewQA` 数据结构验证（检查每个 JD 的三档是否有≥10 组配对且字段完整）

## v1.3 (2026-04-13)

### 新功能
- **同事催促消息**：假关闭按钮触发的同事弹窗，4 种 HR 风格各 8 条独立消息 + 默认兜底 8 条
  - 大厂风格（男成熟）、创业公司风格（男阳光）、外企风格（女成熟）、年轻互联网风格（女阳光）
  - 每种风格含 2 个实习生角色，融入职场梗和真人语感

### Bug 修复
- **安卓版本号显示 1.0**：`build.gradle` 的 `versionCode` 和 `versionName` 从未更新过。修复为 versionCode=4, versionName="1.3"

### 文档
- `APK_BUILD_GUIDE.md` 新增"更新版本号"步骤，提醒每次发版递增 versionCode

## v1.2 (2026-04-12)

### 新功能
- **HR 风格回应系统**：面试官对候选人回答的回应按 good/mid/bad 三档分组，匹配候选人回答质量
- **评分教程**：首次进入反馈页时展示半星/整星点击教程动画
- **HRD 评分对比**：信件场景右上角弹出框对比玩家评分与实际理想评分

### 优化
- 面试回应选择器从扁平数组改为分档对象结构
- 候选人提问环节支持跳过（HR 说辞匹配风格）

## v1.1 (2026-04-10)

### 新功能
- **本地模式**：完整离线可玩，3 个预设 JD 各配备候选人、面试题、回答、谈薪对话、HRD 信件
- **APK 打包**：Capacitor 8 + Android 构建流程，内置 API Key 免配置
- **强制毕业机制**：全拒简历、连续差评、优质候选人 offer 被拒 2 次等触发提前结束

### 工具链
- `pre-build-check.sh` 预检脚本（语法/全局变量/数据结构/环境同步/编码安全/资源完整 6 层检查）
- `APK_BUILD_GUIDE.md` 打包文档

## v1.0 (2026-04-08)

- 初始发布
- 完整游戏流程：引导 → 菜单 → 加载 → 简历筛选 → 面试 → 反馈 → 谈薪 → HRD 评审 → 结算
